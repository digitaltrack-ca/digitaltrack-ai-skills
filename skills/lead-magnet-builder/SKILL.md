# Lead Magnet Builder

## Goal

Create a concise, source-grounded lead magnet that solves one practical problem for a specific audience. Produce a polished DOCX first, then a PDF when reliable conversion is available.

## Portability

- Use any available AI model for synthesis and drafting.
- Use any transcript, file-reading, web, or document tools available.
- Never depend on a model-specific connector when a local-file fallback exists.
- Keep the workflow in this file and use bundled assets for consistent execution.

## Workflow

### 1. Inspect every source

Read all supplied files, URLs, transcripts, notes, or recording summaries.

Extract the audience, urgent problem, desired outcome, practical lessons, examples, proof, claims requiring verification, and private details to remove.

Separate facts, speaker opinions, user interpretation, and inference. Never present a paraphrase as a direct quote.

### 2. Choose the smallest useful asset

Prefer one clear job over a topic dump. Good formats include a field guide, checklist, workbook, diagnostic scorecard, implementation roadmap, mistakes guide, or swipe file.

Default to 5-10 pages. Read `references/content-strategy.md` when choosing the angle, title, structure, CTA, or bilingual approach.

### 3. Resolve the brand

If the user names a brand:

1. Inspect its official brand repository or guidelines.
2. Use confirmed logo, colors, typography, voice, CTA, and contact details.
3. Treat uncertain brand claims as unknown.

For DigitalTrack:

- Read `references/digitaltrack-brand.md`.
- Use the bundled logos.
- Duplicate `assets/DigitalTrack_Bilingual_Lead_Magnet_Template.docx` as the visual starting point; never overwrite the original.

For another brand, replace every DigitalTrack element with confirmed brand information. Omit unavailable logos rather than inventing artwork.

### 4. Design the content

Use this structure:

1. Outcome-driven cover
2. Who the guide is for and what it solves
3. Three to seven practical principles
4. Audience-relevant examples
5. Checklist, worksheet, scorecard, or action plan
6. One clear next step
7. Source note and disclaimer

For bilingual assets, use complete language sections instead of alternating every paragraph. Preserve meaning; do not translate mechanically.

### 5. Draft and build

- Keep titles specific and believable.
- Keep paragraphs short.
- Include at least one actionable checklist.
- Use callouts sparingly.
- Include a source note.
- Do not add unverified facts, quotes, or URLs.
- Preserve the template's typography, color system, logo proportions, margins, headers, footers, and CTA styling.

### 6. Verify before delivery

Check that the DOCX opens, no placeholders remain, the logo is not distorted, every language section is complete, contact details are current, and the source note distinguishes synthesis from quotation.

Render and inspect every page when possible. If rendering is unavailable, use Word or LibreOffice pagination plus structural checks and disclose that image-based QA was not completed.

### 7. Create outreach derivatives after approval

After the user approves the guide, offer a two-email delivery sequence, text/DM, social posts, landing-page copy, or CRM nurture plan.

Do not send, publish, upload, or enroll contacts without explicit approval.

## Rules

- Lead with education, not a disguised pitch.
- Do not invent results, guarantees, statistics, quotes, affiliations, or endorsements.
- Remove unnecessary private transcript details.
- Prefer a narrow problem with a usable action plan.
- Keep the CTA proportional.
- Preserve editable source files.
- Never overwrite an approved template or deliverable without permission.
- Use the active brand name exactly as specified.

## Self-Improvement Loop

1. Note where the agent guessed or searched repeatedly.
2. Update a reference or reusable asset instead of adding one-off instructions.
3. Save brand rules only when confirmed.
4. Test English-only, Spanish-only, and bilingual outputs.
5. After three real uses, review reader feedback and CTA performance, then tighten the workflow.
